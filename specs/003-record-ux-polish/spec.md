# Feature Specification: 記録アプリ UX 拡充

**Feature Branch**: `003-record-ux-polish`
**Created**: 2026-03-12
**Status**: Draft
**Input**: User description: "記録アプリとして足りない機能を拡充させたい。・編集画面にて画像を一覧サムネイルと同じく縦長で表示。 ・編集時にサムネイルが非表示になるため表示(再撮影もできるように) ・ヘッダの戻るボタンの挙動を統一感を出したい(ホームに戻るのか、1つ前に戻るのか、それぞれボタンを配置してもよい) ・一覧画面からタップした先の詳細画面に入力情報があまり表示されない。入力情報はすべて表示したい ・設定ボタンから、"プライバシーポリシー"、"ライセンス"の表示を行いたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 詳細画面で全入力情報を確認する (Priority: P1)

ユーザーが一覧画面からノートをタップしたとき、詳細画面に記録したすべての情報（銘柄名、外観、香り、味わい、余韻、メモ、評価、画像）が表示される。現状では一部の情報しか見えないため、記録を振り返る用途として機能しない。

**Why this priority**: ノートアプリの根本的な価値である「記録した情報を確認できること」が損なわれている。最も利用頻度が高いフローであり、他の改善よりも先に解決すべき。

**Independent Test**: 入力情報を持つノートを選択し、詳細画面で全フィールドが表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** テイスティングノートが外観・香り・味わい・余韻・メモ・評価・画像すべて入力済みの状態で、**When** 一覧画面からそのノートをタップする、**Then** 詳細画面に全フィールドの内容が表示される
2. **Given** テイスティングノートの一部フィールドが未入力の状態で、**When** 詳細画面を開く、**Then** 未入力フィールドは表示されない
3. **Given** 長いテキストが入力されたノートで、**When** 詳細画面を開く、**Then** テキストが省略されずすべて表示される

---

### User Story 2 - 編集画面で画像を縦長表示・再撮影する (Priority: P2)

ユーザーが編集画面を開いたとき、ボトルの画像が一覧サムネイルと同じ縦長（ポートレート）形式で表示される。また、既存の画像を確認しながら再撮影または差し替えができる。現状では編集中に画像が見えないため、どの画像を登録しているか確認できない。

**Why this priority**: 画像の確認・変更はユーザーの記録品質に直結する。特に「どのボトルか」を視覚的に確認しながら編集できることは重要。詳細画面の修正より開発コストが高いが、ユーザー体験への影響が大きい。

**Independent Test**: 画像つきノートの編集画面を開き、縦長サムネイルが表示されること、および画像変更操作が完了することを確認できる。

**Acceptance Scenarios**:

1. **Given** 画像が登録されているノートの編集画面を開いたとき、**When** 画面を表示する、**Then** 縦長（2:3 比率）のサムネイルが表示される
2. **Given** 編集画面に縦長サムネイルが表示されている状態で、**When** ユーザーが再撮影・画像変更ボタンをタップする、**Then** カメラまたは画像選択が起動し、新しい画像に差し替えられる
3. **Given** 画像が未登録のノートの編集画面を開いたとき、**When** 画面を表示する、**Then** 縦長プレースホルダーと画像追加ボタンが表示される

---

### User Story 3 - ナビゲーション戻るボタンの挙動を統一する (Priority: P3)

ユーザーがアプリ内を移動するとき、各画面のヘッダにある「戻る」操作が一貫したルールに従って動作する。「1つ前の画面に戻る」「ホームに戻る」の区別が明示され、ユーザーが迷わず操作できる。

**Why this priority**: ナビゲーションの一貫性はアプリの使いやすさの基盤だが、機能的な情報損失はなく既存機能の改善に留まるため P3 とする。

**Independent Test**: 各画面（詳細、編集、新規作成）でナビゲーションボタンをタップし、意図した画面に遷移することを確認できる。

**Acceptance Scenarios**:

1. **Given** 一覧 → 詳細画面の遷移後に、**When** 戻るボタンをタップする、**Then** 一覧画面に戻る
2. **Given** 詳細 → 編集画面の遷移後に、**When** 戻るボタンをタップする、**Then** 詳細画面に戻る（変更がある場合は破棄確認を表示）
3. **Given** 新規作成画面で、**When** 戻るボタンをタップする、**Then** 一覧画面（ホーム）に戻る
4. **Given** どの画面でも、**When** ホームアイコンをタップする、**Then** 一覧画面に直接戻る

---

### User Story 4 - 設定画面からプライバシーポリシーとライセンスを確認する (Priority: P4)

ユーザーが設定画面からプライバシーポリシーおよびライセンス情報を確認できる。アプリの公開に際して法的要件を満たすために必要な機能。

**Why this priority**: 機能的な優先度は低いが、公開アプリとして必須の法的要件。UX への影響は小さいが存在しないと公開要件を満たせない。

**Independent Test**: 設定画面からプライバシーポリシーおよびライセンス画面を開き、内容が表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 設定画面を開いた状態で、**When** 「プライバシーポリシー」をタップする、**Then** プライバシーポリシーの内容が画面に表示される
2. **Given** 設定画面を開いた状態で、**When** 「ライセンス」をタップする、**Then** 使用ライブラリのライセンス一覧が表示される
3. **Given** プライバシーポリシー / ライセンス表示中に、**When** 戻るボタンをタップする、**Then** 設定画面に戻る

---

### Edge Cases

- 編集画面で画像を削除（画像なし状態に戻す）した場合、プレースホルダーに戻り、保存後は画像なしノートになる
- ライセンス情報が大量になった場合のスクロール対応が必要
- 編集中に画像変更後にキャンセルした場合、元の画像が維持される
- 新規作成画面（保存前）でバックナビした場合、入力中データの破棄確認ダイアログを表示する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 詳細画面はノートに記録された全フィールド（銘柄名・蒸留所・ヴィンテージ・外観・香り・味わい・余韻・メモ・評価・ボトル画像・記録日時）を表示しなければならない
- **FR-001b**: 詳細画面のボトル画像は縦長（2:3 比率）で表示しなければならない
- **FR-002**: 詳細画面において未入力のフィールドは表示しない
- **FR-003**: 編集画面はボトル画像を縦長（2:3 比率）のサムネイルとして表示しなければならない
- **FR-004**: 編集画面のサムネイルには画像変更（再撮影・ライブラリ選択）を起動するボタンを配置しなければならない
- **FR-005**: 画像未登録のノートの編集画面では縦長プレースホルダーと画像追加ボタンを表示しなければならない
- **FR-005b**: 画像登録済みのノートの編集画面では、画像変更ボタンに加えて画像削除ボタンを表示しなければならない。削除後はプレースホルダー表示に戻る
- **FR-006**: 各画面のヘッダ左端に「← 戻る」ボタン（1つ前の画面に戻る）、右端にホームアイコンボタン（一覧画面に直接戻る）を配置しなければならない。一覧画面自体ではホームアイコンを非表示にする
- **FR-007**: 編集・新規作成画面でナビゲーション操作をした場合、未保存の変更があれば破棄確認ダイアログを表示しなければならない。ユーザーが「破棄する」を選択した場合、操作起点に応じて遷移する（「← 戻る」起点 → 1つ前の画面、ホームアイコン起点 → 一覧画面）
- **FR-008**: 設定画面に「プライバシーポリシー」と「ライセンス」へのナビゲーション項目を追加しなければならない
- **FR-009**: プライバシーポリシー画面にはアプリのデータ取り扱い方針（ローカル保存のみ・外部送信なし）を明記しなければならない
- **FR-010**: ライセンス画面には使用しているオープンソースライブラリとそのライセンス情報を表示しなければならない

### Key Entities

- **TastingNote（テイスティングノート）**: 銘柄名・蒸留所・ヴィンテージ・外観・香り・味わい・余韻・メモ・評価・ボトル画像・記録日時を持つ中心エンティティ。詳細画面はこのエンティティの全フィールドを表示する
- **PrivacyPolicy（プライバシーポリシー）**: アプリ内に静的コンテンツとして保持。データ収集なし・ローカル保存のみを明記
- **LicenseInfo（ライセンス情報）**: 使用ライブラリ名・バージョン・ライセンス種別・著作権表示のリスト

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 詳細画面を開いたとき、記録されている全フィールドがスクロールにより漏れなく確認できる（情報欠落ゼロ）
- **SC-002**: 編集画面でボトル画像が即座に表示され、ユーザーが 2 タップ以内に画像変更を開始できる
- **SC-003**: ナビゲーション操作において、ユーザーが意図した画面に毎回正確に遷移できる（誤遷移ゼロ）
- **SC-004**: 設定画面からプライバシーポリシーおよびライセンス情報に 1 タップでアクセスできる
- **SC-005**: 未保存の変更がある状態でナビゲーションしたとき、100% の確率で破棄確認が表示される

## Clarifications

### Session 2026-03-12

- Q: ナビゲーションヘッダのボタン配置はどうするか？ → A: ヘッダ左に「← 戻る」（1つ前の画面）、右にホームアイコン（一覧直帰）を配置
- Q: 編集画面での画像削除は今回スコープに含むか？ → A: 含める。編集画面に画像削除ボタンを追加する
- Q: 変更破棄確認後の遷移先はどこか？ → A: 起点に従う。「← 戻る」起点なら1つ前の画面、ホームアイコン起点なら一覧画面に遷移

## Assumptions

- プライバシーポリシーの文面は開発者が別途用意し、アプリ内に静的テキストとして組み込む（外部 URL へのリンクは不要）
- ライセンス情報はコード内に定数として管理する
- 「ホームに戻る」ボタンは全画面共通のヘッダコンポーネントに配置する
- 画像変更時は既存の圧縮・500KB 上限ルールを引き続き適用する
- 詳細画面の画像表示も縦長（2:3 比率）に統一する
