# Feature Specification: ノート一覧レイアウト変更（縦長サムネイル・1列表示）

**Feature Branch**: `002-note-list-portrait-layout`
**Created**: 2026-03-12
**Status**: Clarified
**Input**: User description: "サムネイル画像は主にウイスキーボトルの画像なので、縦長で表示させたい。一覧画面の並びも1列1件で左に画像、右にノートのプレビューで"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ノート一覧を縦長サムネイル＋横並びレイアウトで表示 (Priority: P1)

ウィスキーボトルの画像は縦長の形状が多いため、現在の正方形サムネイルでは構図が崩れる。ユーザーが一覧画面を開いたとき、各ノートが「左側に縦長サムネイル、右側に銘柄名・評価・プレビューテキスト」の横並び1行レイアウトで表示されることで、ボトル画像を正しい縦長比率で確認しながら目的のノートをすばやく見つけられる。

**Why this priority**: 画像の表示品質が直接ユーザー体験に影響する中核の変更。これ単体で完結し、他の機能への依存はない。

**Independent Test**: ノートが複数存在する状態で一覧画面を開き、各ノートが縦長サムネイル付きの1行レイアウトで表示されることで独立して検証可能。

**Acceptance Scenarios**:

1. **Given** ノートが存在する状態で、**When** 一覧画面を表示する、**Then** 各ノートが1行1件で表示され、左側に縦長比率のサムネイルエリア、右側に銘柄名・評価・コメントプレビューが配置される
2. **Given** 画像が登録されたノートがある状態で、**When** 一覧画面を表示する、**Then** ボトル画像が縦長比率（縦:横 = 3:2 程度）でサムネイルとして表示される
3. **Given** 画像が未登録のノートがある状態で、**When** 一覧画面を表示する、**Then** 画像エリアにプレースホルダー（アイコン等）が縦長比率で表示される
4. **Given** テイスティングコメント（外観・香り・味わい・余韻のいずれか）が入力済みのノートがある状態で、**When** 一覧画面を表示する、**Then** 右側のプレビューエリアに最初の入力済みコメントの冒頭テキストが表示される
5. **Given** テイスティングコメントが未入力のノートがある状態で、**When** 一覧画面を表示する、**Then** プレビューエリアは空白またはプレースホルダーで表示され、エラーにならない
6. **Given** 長い銘柄名のノートがある状態で、**When** 一覧画面を表示する、**Then** 銘柄名は1行に収まる範囲で省略表示（末尾「…」）され、レイアウトが崩れない
7. **Given** ノートをタップする、**When** 一覧画面で任意のノート行をタップする、**Then** 該当ノートの詳細画面へ遷移する

---

### Edge Cases

- 画像未登録のノートと画像登録済みのノートが混在する場合、統一されたレイアウトで表示されるか？
- 非常に長いテイスティングコメントが入力されている場合、プレビュー文字列が適切に省略されるか？
- 500件以上のノートが存在する場合、1列レイアウトでのスクロールパフォーマンスは維持されるか？
- ノートが0件の場合、既存の空状態メッセージが正しく表示されるか？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ノート一覧画面は、1行に1件のノートを表示する縦1列レイアウトを採用しなければならない（現行の2列グリッドを置き換える）
- **FR-002**: 各行の左側にサムネイル画像エリアを配置し、縦長比率（縦:横 = 3:2）で表示しなければならない
- **FR-003**: 画像が登録されているノートは、ボトル画像をサムネイルエリアに縦長比率を保ちながら表示しなければならない
- **FR-004**: 画像が未登録のノートは、サムネイルエリアにプレースホルダーを縦長比率で配置しなければならない
- **FR-005**: 各行の右側に、銘柄名・総合評価・テイスティングコメントプレビュー・記録日時を表示しなければならない。総合評価は数値のみ（例: `85`）で表示し、未入力の場合は非表示とする。記録日時は絶対日付形式（例: `2026/03/12`）で表示する
- **FR-006**: テイスティングコメントプレビューは、外観・香り・味わい・余韻・メモの順で最初の入力済みフィールドの冒頭テキストを表示しなければならない
- **FR-007**: 銘柄名は1行省略表示（末尾「…」）とし、プレビューテキストは最大2行で省略しなければならない
- **FR-008**: ノートをタップすると詳細画面へ遷移する動作は現行通り維持しなければならない
- **FR-009**: ノートが0件の場合の空状態表示は現行通り維持しなければならない

### Key Entities

- **NoteListItem（一覧表示項目）**: TastingNote の表示用ビュー。左エリア: 縦長サムネイル（またはプレースホルダー）。右エリア: 銘柄名（省略あり）・総合評価・コメントプレビュー（最初の入力済みフィールド冒頭）・記録日時

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 一覧画面でボトル画像が縦長比率で表示され、ボトルの全体的な形状が視認できる
- **SC-002**: 画像あり・なしが混在する一覧でもレイアウト崩れが発生しない
- **SC-003**: 500件のノートが存在する状態での一覧表示が1.5秒以内に完了する（既存パフォーマンス要件を維持）

## Clarifications

### Session 2026-03-12

- Q: 総合評価の表示形式は？ → A: 数値のみ（例: `85`）、未入力は非表示
- Q: 記録日時のフォーマットは？ → A: 絶対日付形式（例: `2026/03/12`）

## Assumptions

- 画像の縦横比は変更せず、`object-fit: contain` 相当（余白付き）で表示し、ボトル全体が見えることを優先する
- プレビューに表示するコメントフィールドの優先順位は「外観 → 香り → 味わい → 余韻 → メモ」とする
- 詳細画面・編集画面・画像登録機能のレイアウトはこのfeatureのスコープ外とする
- 既存のサムネイル画像データはそのまま利用し、再圧縮・再保存は行わない
