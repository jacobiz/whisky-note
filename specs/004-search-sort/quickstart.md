# Quickstart / Integration Scenarios: 検索・ソート機能

**Feature Branch**: `004-search-sort`
**Date**: 2026-03-12

---

## シナリオ 1: テキスト検索でノートを絞り込む（US1）

**前提**: 3件のノート（「山崎12年」「白州18年」「Yamazaki 18」）が存在する

```
操作: SearchSortBar の検索バーに「山崎」と入力
期待: 「山崎12年」のみ表示される
      「白州18年」「Yamazaki 18」は非表示

操作: 「yamazaki」と入力（小文字・英字）
期待: 「Yamazaki 18」のみ表示（大文字小文字を区別しない）

操作: × ボタンをタップ
期待: 3件すべてが再表示される
```

---

## シナリオ 2: 検索0件時の専用メッセージ（US1）

**前提**: 「山崎12年」1件のみ存在する

```
操作: 「羽生」と入力（一致なし）
期待: 「該当するノートが見つかりません」メッセージが表示される
     「検索条件を変えてみてください」サブテキストが表示される
     🥃 アイコンと「テイスティングノートがまだありません」は表示されない
```

---

## シナリオ 3: ソートの切り替え（US2）

**前提**: 評価が異なる3件（評価90「A」、評価75「B」、評価未設定「C」）が存在する

```
操作: ソートを「評価が高い順」に変更
期待: A（90）→ B（75）→ C（未設定）の順で表示される

操作: ソートを「銘柄名順」に変更
期待: アルファベット・五十音順で表示される
```

---

## シナリオ 4: 検索+ソートの同時適用（US1 + US2）

**前提**: 「山崎12年（評価80）」「山崎18年（評価95）」「白州12年（評価85）」が存在する

```
操作: 「山崎」で検索 → 「評価が高い順」でソート
期待: 「山崎18年（95）」→「山崎12年（80）」の2件が表示される
     白州12年は表示されない
```

---

## シナリオ 5: ページ遷移後の状態保持（FR-011, FR-012）

```
操作: 「山崎」で検索し「評価が高い順」でソートした状態でノート詳細へ遷移
操作: ブラウザバックで一覧に戻る
期待: 「山崎」の検索クエリが復元されている
     「評価が高い順」ソートが復元されている
     絞り込まれた件数のままで表示される
```
