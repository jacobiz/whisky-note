# Feature Specification: ノートリストの検索・ソート機能

**Feature Branch**: `004-search-sort`
**Created**: 2026-03-12
**Status**: Draft
**Input**: User description: "検索・ソート機能を"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - テキスト検索によるノート絞り込み (Priority: P1)

ユーザーは銘柄名・蒸留所名でテイスティングノートを検索し、目的のノートをすばやく見つけることができる。コレクションが増えるにつれ、全件スクロールで探すのが困難になるため、検索は最も優先度の高い機能である。

**Why this priority**: コレクションが多くなるほど目的のノートを探すのが困難になり、ユーザーの日常的な操作に直接影響する最重要機能。

**Independent Test**: 銘柄名でテキスト入力し、一致するノートだけが表示されることを確認できる。検索クリア後は全件に戻ることで完結する。

**Acceptance Scenarios**:

1. **Given** ノートリストに複数のノートが存在する, **When** 検索バーに銘柄名の一部を入力する, **Then** 銘柄名または蒸留所名に入力文字列を含むノートのみが表示される
2. **Given** 検索バーに文字が入力されている, **When** 検索バーをクリア（×ボタンまたは全削除）する, **Then** 全ノートが再表示される
3. **Given** 検索条件に一致するノートが存在しない, **When** 検索を実行する, **Then** 「該当するノートが見つかりません」というメッセージが表示される
4. **Given** ノートリストが表示されている, **When** 検索バーに文字を入力する, **Then** 追加のボタン操作なしにリアルタイムでリストが更新される

---

### User Story 2 - ノートリストのソート (Priority: P2)

ユーザーは好みの順序でノートを並び替えて閲覧できる。デフォルトは記録日時の新しい順とし、評価順・名前順などに切り替えられる。

**Why this priority**: 検索でノートを絞り込んだ後や、高評価ノートを一覧したいときに必要な補助的な機能。

**Independent Test**: ソートセレクターで「評価の高い順」を選択し、評価スコアの高いノートから順に表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** ノートリストが表示されている, **When** ソートを「記録日時：新しい順」に設定する, **Then** 最新のノートが先頭に表示される（デフォルト動作）
2. **Given** ノートリストが表示されている, **When** ソートを「記録日時：古い順」に設定する, **Then** 最も古いノートが先頭に表示される
3. **Given** ノートリストが表示されている, **When** ソートを「評価：高い順」に設定する, **Then** 評価スコアが高いノートから順に表示され、評価未入力のノートは末尾に表示される
4. **Given** ノートリストが表示されている, **When** ソートを「銘柄名：A-Z / あ行順」に設定する, **Then** 銘柄名がアルファベット・五十音順で表示される
5. **Given** 検索条件が入力されている状態, **When** ソートを変更する, **Then** 絞り込まれた結果内でソートが適用される

---

### Edge Cases

- 検索文字列が空の場合はフィルタリングを行わず全件表示する
- 評価未入力（undefined / null）のノートを「評価：高い順」でソートする場合、評価済みノートの後に表示する
- 大文字・小文字を区別しない検索（例: "yamazaki" で「Yamazaki」がヒットする）
- 1件のみのリストでも検索・ソートは正常に動作する
- 0件のリストで検索・ソートを操作してもエラーにならない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 検索バーはノートリスト画面の上部に常に表示される
- **FR-002**: 検索は銘柄名（brandName）および蒸留所名（distillery）を対象とする
- **FR-003**: 検索はリアルタイム（入力のたびに即時反映）で動作する
- **FR-004**: 検索は大文字・小文字を区別しない
- **FR-005**: 検索一致判定は部分一致とする
- **FR-006**: 検索結果が0件の場合は「ノートがまだない」状態とは別の専用メッセージ・UIを表示する（例：「該当するノートが見つかりません」＋検索条件変更を促すサブテキスト）
- **FR-007**: 検索バーにはクリアボタン（×）を設け、タップ一操作で検索をリセットできる
- **FR-008**: ソートセレクターはノートリスト画面に設置する（検索バーと同一エリアが望ましい）。UIコンポーネントはネイティブ `<select>` 要素を使用する（OS標準ピッカーによりモバイル操作性を確保）
- **FR-009**: 利用可能なソートオプションは以下の4種類とする：
  - 記録日時：新しい順（デフォルト）
  - 記録日時：古い順
  - 評価：高い順
  - 銘柄名：A-Z / あ行順
- **FR-010**: 検索とソートは同時に適用できる（検索で絞り込んだ結果をソートする）
- **FR-011**: 選択中のソートオプションはセッション中に保持される（ノート詳細から戻っても維持）
- **FR-012**: 検索キーワードはセッション中に保持される（ノート詳細から戻っても維持）

### Key Entities

- **TastingNote（既存）**: brandName（銘柄名）、distillery（蒸留所名）、rating（評価）、createdAt（記録日時）を検索・ソートの対象フィールドとして利用する
- **SearchSortState**: 現在の検索キーワードとソートオプションを管理する状態。ページ遷移後も保持される（セッション中のみ）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 検索バーへの入力から100ms以内にリストが更新される（体感的に即時反映）
- **SC-002**: 10件以上のノートがある状態で、目的のノートを3秒以内に見つけられる
- **SC-003**: 検索バーとソートセレクターはスマートフォン（375px幅）で快適に操作できる（タップターゲット44px以上）
- **SC-004**: 検索・ソート操作が原因でアプリがクラッシュ・フリーズしないこと
- **SC-005**: ノート詳細画面から戻ったとき、直前の検索・ソート状態が復元される

## Clarifications

### Session 2026-03-12

- Q: ソートセレクターのUIコンポーネントはどの種類か？ → A: ネイティブ `<select>` 要素（OS標準ピッカー）
- Q: 検索0件時の空状態は既存の「ノートなし」UIと同じか別か？ → A: 別メッセージ・UIで区別する（2つのシナリオを独立して表示）

## Assumptions

- 検索対象は銘柄名と蒸留所名のみ（nose・palate等のテイスティングフィールドは対象外）
- 全文検索・あいまい検索（Fuzzy）は対象外（単純な部分一致で十分）
- ソートオプションはシングル選択のみ（複合ソートは対象外）
- ノート件数が数千件になることを想定し、クライアントサイドフィルタリングで十分とする
- 検索・ソート状態はPiniaストアに保存し、ページリロード後はデフォルトに戻る（永続化は対象外）
