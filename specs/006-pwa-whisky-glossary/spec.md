# Feature Specification: PWA強化 & ウィスキー用語辞典

**Feature Branch**: `006-pwa-whisky-glossary`
**Created**: 2026-03-12
**Status**: Draft
**Input**: User description: "PWA強化（オフラインキャッシュ + インストール促進）とウィスキー辞書/フレーバー用語集"

## Clarifications

### Session 2026-03-12

- Q: インストール促進の表示方法は？（アプリ内バナー + 設定ボタン vs 設定ボタンのみ） → A: 設定画面ボタンのみ（バナーなし）
- Q: ノートフォームからの用語辞典アクセス方式は？ → A: モーダルシート（画面下からスライドアップ）、閉じるとフォームに戻る
- Q: Service Worker 更新検知後のユーザー通知方法は？ → A: トースト通知「新しいバージョンが利用可能です」を表示し、ユーザーがタップしてリロード（自動リロードなし）
- Q: 用語辞典のメインナビゲーション配置は？ → A: ホーム画面にカードボタンとして配置（一覧・新規作成と並列）
- Q: 用語詳細の表示形式は？ → A: 一覧内アコーディオン展開（詳細専用画面なし）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - アプリをホーム画面にインストールする (Priority: P1)

ユーザーがブラウザでアプリにアクセスしたとき、「ホーム画面に追加」のプロンプトまたは案内が表示され、ネイティブアプリのようにインストールできる。インストール後はスタンドアロンモードで起動し、ブラウザのアドレスバーが表示されない。

**Why this priority**: PWA の根幹機能。インストールされることで継続利用率が高まり、オフライン動作も保証される。Lighthouse PWA スコア ≥90 はプロジェクト憲法の必須要件。

**Independent Test**: アプリにアクセスしてインストールプロンプトを確認し、インストール後にスタンドアロンモードで起動することを確認できる。

**Acceptance Scenarios**:

1. **Given** ブラウザでアプリにアクセスした状態で、**When** ブラウザのインストール基準（manifest + Service Worker）を満たしている、**Then** ブラウザのネイティブインストールプロンプトが表示される
2. **Given** インストールプロンプトが表示された状態で、**When** ユーザーが「インストール」を選択する、**Then** アプリがホーム画面・アプリランチャーに追加され、以降はスタンドアロンモードで起動する
3. **Given** アプリがインストールされた状態で、**When** ホーム画面から起動する、**Then** ブラウザのアドレスバーが表示されず、全画面のアプリ体験が得られる
4. **Given** アプリがインストール可能な状態で、**When** ユーザーが設定画面を開く、**Then** 「ホーム画面に追加」ボタンが表示され、タップするとインストールプロンプトが起動する
5. **Given** アプリが既にインストールされている状態で、**When** 設定画面を開く、**Then** 「ホーム画面に追加」ボタンは表示されないか無効化される

---

### User Story 2 - ネットワーク未接続でもアプリを使用する (Priority: P1)

ユーザーがオフライン状態（機内モード・電波圏外など）でアプリを起動・操作しても、すべての機能（ノート閲覧・作成・編集・削除・辞典参照）が正常に動作する。

**Why this priority**: プロジェクト憲法「ローカルファースト」の根幹要件。テイスティング現場（バー・試飲会）はネットワーク環境が不安定なケースがある。

**Independent Test**: 機内モードでアプリを開き、既存ノートの閲覧・新規作成・辞典参照が動作することで独立検証可能。

**Acceptance Scenarios**:

1. **Given** アプリが一度でもオンラインでアクセスされた状態で、**When** ネットワーク接続を切断してアプリを開く、**Then** アプリが正常に起動し全画面が表示される（白画面やエラー画面にならない）
2. **Given** オフライン状態で、**When** ノート一覧・詳細・作成・編集の各画面に遷移する、**Then** すべての画面が正常に表示・操作できる
3. **Given** オフライン状態で、**When** 用語辞典を開いてキーワードを検索する、**Then** キャッシュ済みの辞典データが表示される
4. **Given** アプリのリソースが更新された状態で、**When** オンラインでアプリを起動する、**Then** バックグラウンドで新しいバージョンがキャッシュされ、次回起動時に最新版が適用される
5. **Given** Service Worker のキャッシュが消去された状態で、**When** オンラインでアプリにアクセスする、**Then** 自動的に再キャッシュされ通常通り動作する

### User Story 3 - ウィスキー用語をテイスティング中に調べる (Priority: P2)

ユーザーがテイスティング中にフレーバー用語（「スモーキー」「シェリー樽」「フィニッシュ」など）の意味や使い方を、アプリ内で素早く調べられる。外観・香り・味わい・余韻のカテゴリ別に用語が整理されており、初心者がテイスティングコメントを書く際の参考にできる。

**Why this priority**: テイスティングノートの質を高める補助機能。すでに `docs/whisky-tasting.md` に知識ベースが存在しており、アプリ内で参照可能にするだけで大きな価値を提供できる。

**Independent Test**: 辞典画面を開き、「スモーキー」などのキーワードで検索して説明が表示されることで独立検証可能。

**Acceptance Scenarios**:

1. **Given** アプリを起動した状態で、**When** ホーム画面の「用語辞典」カードボタンをタップする、**Then** テイスティング用語の一覧画面が表示される
2. **Given** 用語辞典画面で、**When** カテゴリ（外観・香り・味わい・余韻・その他）を選択する、**Then** そのカテゴリに属する用語一覧がフィルタリングされて表示される
3. **Given** 用語辞典画面で、**When** 検索ボックスにキーワードを入力する、**Then** 用語名・説明に一致する用語がリアルタイムで絞り込まれる
4. **Given** 用語一覧で、**When** ひとつの用語をタップする、**Then** その行がアコーディオン展開され、日本語と英語の名称・説明・使用例がインラインで表示される（詳細専用画面への遷移はない）
5. **Given** ノート作成・編集画面で、**When** テイスティングコメントを入力中に「用語辞典を参照」ボタンをタップする、**Then** 辞典が画面下からモーダルシートとしてスライドアップして開き、閉じると元のフォームに戻る（入力内容は保持される）
6. **Given** 用語辞典を日本語モードで利用中に、**When** アプリ言語を英語に切り替える、**Then** 用語の説明文・UI ラベルが英語で表示される

---

### Edge Cases

- インストールプロンプトを一度「後で」と閉じたユーザーが再度インストールしたい場合、設定画面から再度促せるか？
- iOS Safari では Web App Manifest のインストール挙動がブラウザによって異なるが、代替手順を案内できるか？
- Service Worker のアップデート中にユーザーが操作していた場合、データ整合性は保たれるか？
- 辞典データが 200 件以上になった場合、検索のレスポンスは許容範囲内か？
- 用語辞典をオフラインで開こうとしたが初回キャッシュが完了していない場合、適切なメッセージが表示されるか？

## Requirements *(mandatory)*

### Functional Requirements

**PWA 強化**

- **FR-001**: アプリは Web App Manifest を持ち、アプリ名・アイコン（192px / 512px）・テーマカラー・背景色・スタンドアロン表示モードを定義しなければならない
- **FR-002**: Service Worker はアプリシェル（HTML・CSS・JS・フォント・アイコン）を初回アクセス時にキャッシュし、以降はキャッシュから配信しなければならない（Cache First 戦略）
- **FR-003**: Service Worker は静的アセットのキャッシュ更新をバックグラウンドで行い、新バージョンの準備完了時にトースト通知「新しいバージョンが利用可能です」を画面下部に表示しなければならない。ユーザーがトーストをタップした場合のみページをリロードし、自動リロードは行わない
- **FR-004**: ネットワーク未接続の状態でも、アプリの全画面（ホーム・一覧・作成・編集・詳細・設定・用語辞典）が正常に動作しなければならない
- **FR-005**: インストール可能な条件が整った際に、設定画面に「ホーム画面に追加」ボタンを表示しなければならない。アプリ内バナーによるプロンプトは表示しない
- **FR-006**: アプリは Lighthouse PWA 監査でスコア 90 以上を達成しなければならない

**ウィスキー用語辞典**

- **FR-007**: アプリはホーム画面のカードボタン（ノート一覧・新規作成と並列）からアクセス可能な用語辞典画面を提供しなければならない
- **FR-008**: 用語辞典は以下の7カテゴリによる分類を持たなければならない: テイスティングプロセス・フレーバーホイール・マウスフィール・フィニッシュ・製造／熟成・産地・一般。UI上のカテゴリタブは「すべて」を含む8項目として表示する
- **FR-009**: 各用語は、日本語名・英語名・説明文（日本語）・説明文（英語）・使用例を持たなければならない
- **FR-010**: ユーザーは用語名または説明文に対してキーワード検索でき、結果はリアルタイムで絞り込まれなければならない
- **FR-011**: 用語辞典のデータはアプリバンドルに含まれ、ネットワーク接続なしで全件参照できなければならない
- **FR-012**: ノート作成・編集フォームから用語辞典をモーダルシート（画面下からスライドアップ）で開けるボタンを提供しなければならない。モーダルを閉じると元のフォームに戻り、入力内容は保持されなければならない
- **FR-013**: 用語辞典の表示言語はアプリ全体の言語設定に連動しなければならない

### Key Entities

- **GlossaryEntry（用語エントリ）**: 辞典の1用語。属性: 用語ID・日本語名・英語名・カテゴリ（外観/香り/味わい/余韻/その他）・説明文（日本語）・説明文（英語）・使用例（日本語）・使用例（英語）
- **GlossaryCategory（カテゴリ）**: 用語の内部分類。7種: `tasting-process`（テイスティングプロセス）、`flavour-wheel`（フレーバーホイール）、`mouthfeel`（マウスフィール）、`finish`（フィニッシュ）、`production`（製造・熟成）、`region`（産地）、`general`（一般）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: アプリに一度アクセスしたユーザーは、ネットワーク未接続の状態でもすべての主要機能（ノート操作・用語辞典）を利用できる
- **SC-002**: Lighthouse PWA 監査スコアが 90 以上を達成する
- **SC-003**: インストール済みのアプリはスタンドアロンモードで起動し、ブラウザ UI が表示されない
- **SC-004**: 用語辞典のキーワード検索が入力から 0.3 秒以内に結果を返す
- **SC-005**: 用語辞典に 50 件以上の用語が収録されており、テイスティングの全フェーズをカバーする
- **SC-006**: ノート作成画面から用語辞典を参照し、フォームに戻るまでの操作が 3 タップ以内で完了する

## Assumptions

- 用語辞典のデータソースは既存の `docs/whisky-tasting.md` および `docs/whisky-tasting-en.md` を基に静的 JSON として整備する
- iOS Safari のインストール（ホーム画面に追加）はブラウザ UI からのみ可能なため、設定画面には iOS 向けの手順案内テキストを表示する
- Service Worker のキャッシュ戦略はアプリシェルに限定し、IndexedDB のデータはキャッシュ対象外（Dexie.js が直接管理）
- 用語辞典は読み取り専用とし、ユーザーによる編集・追加・削除はスコープ外
- 用語辞典は新規ルート（`/glossary`）として実装し、詳細専用ルートは作成しない（詳細はアコーディオン展開で表示）
