# Feature Specification: ウィスキーテイスティングノート記録

**Feature Branch**: `001-tasting-note-record`
**Created**: 2026-03-11
**Status**: Draft
**Input**: User description: "ウイスキーのテイスティングノートを記録する。スマホ端末で撮影したボトル画像をサムネイル画像として登録。日本語英語の多言語対応。高級感のあるデザイン"

## Clarifications

### Session 2026-03-11

- Q: ノート一覧のソート順は？ → A: 記録日時の降順（最新順・固定）
- Q: 「高級感」デザインの方向性は？ → A: ダークテーマ（黒・深みのあるネイビー）+ 金色アクセント
- Q: 画像の圧縮後保存サイズ上限は？ → A: 500KB
- Q: フォーム入力中のバックグラウンド移行時の動作は？ → A: 自動的に下書き保存し、再起動後も復元する
- Q: 外観・香り・味わい・余韻・メモの各テキストフィールドの文字数上限は？ → A: 各フィールド 1,000文字
- Q: 同一銘柄名で複数のノートを作成できるか？ → A: 制約なし（何件でも作成可能）
- Q: 下書きは同時に何件まで存在できるか？ → A: 常に1件のみ（新規作成開始時に前の下書きを上書き）
- Q: ノート削除時に確認ダイアログを表示するか？ → A: 確認ダイアログあり（承認後に削除）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - テイスティングノートの新規作成 (Priority: P1)

ウィスキー愛好家が、テイスティング中にスマートフォンで外観・香り・味わい・余韻などの印象をノートに記録する。記録したノートは後から一覧で振り返ることができる。同じ銘柄を異なる機会にテイスティングした場合も、それぞれ独立したノートとして記録できる。

**Why this priority**: テイスティングノートの記録がこの機能の核心であり、これなしでは他の機能が成立しない。最小限の実装でも単独で価値を提供できる。

**Independent Test**: 銘柄名・評価・テイスティングコメントを入力して保存し、一覧画面に表示されることで独立して検証可能。

**Acceptance Scenarios**:

1. **Given** アプリを開いた状態で、**When** 「新規ノート作成」を選択して銘柄名・外観・香り・味わい・余韻・総合評価（100点満点）・メモを入力し保存する、**Then** 作成したノートがノート一覧に表示され、再度開いたときに入力内容が正確に復元される
2. **Given** ノート作成フォームで、**When** 銘柄名を入力せずに保存しようとする、**Then** エラーメッセージが表示されて保存が阻止される
3. **Given** 保存済みのノートがある状態で、**When** そのノートを開いて内容を編集し保存する、**Then** 更新内容が反映されて一覧にも変更が反映される
4. **Given** 保存済みのノートがある状態で、**When** そのノートの削除を選択する、**Then** 確認ダイアログが表示され、承認すると一覧からそのノートが消える（キャンセルすると削除されない）
5. **Given** ノート作成フォームに入力中に、**When** アプリをバックグラウンドに移して再度開く、**Then** 入力中の内容が復元され、下書きとして継続編集できる
6. **Given** 下書きが存在する状態で、**When** 新たに「新規ノート作成」を開始する、**Then** 前の下書きは破棄され、新しい空のフォームが表示される
7. **Given** 同じ銘柄名（例: "山崎12年"）のノートが既に存在する状態で、**When** 同じ銘柄名で新規ノートを作成する、**Then** 制約なく保存され、一覧に2件表示される

---

### User Story 2 - ボトル画像の登録 (Priority: P2)

ユーザーがスマートフォンのカメラでウィスキーボトルを撮影し、またはカメラロールから選択して、テイスティングノートのサムネイル画像として登録する。

**Why this priority**: 視覚的な識別性を高め、記録の品質と愛着を大幅に向上させる。P1のノート記録機能があれば、画像なしでも基本的な価値は提供できるため P2。

**Independent Test**: ノート作成時に端末カメラまたはフォトライブラリから画像を選択し、登録後のノート一覧・詳細画面にサムネイルが表示されることで検証可能。

**Acceptance Scenarios**:

1. **Given** ノート作成・編集画面で、**When** 画像エリアをタップしてカメラで撮影する、**Then** 撮影した画像がサムネイルとしてノートに紐付けられ、保存後も表示される
2. **Given** ノート作成・編集画面で、**When** 画像エリアをタップしてフォトライブラリから画像を選択する、**Then** 選択した画像がサムネイルとして登録される
3. **Given** 画像が登録されたノートで、**When** 一覧画面を表示する、**Then** 各ノートに対応するサムネイル画像が表示される
4. **Given** ノートに画像が登録されている状態で、**When** 画像を変更または削除する、**Then** 変更または削除が即座に反映される
5. **Given** 500KBを超えるサイズの画像を選択したとき、**When** 登録しようとする、**Then** 画像が自動的に500KB以下に圧縮されてから保存される
6. **Given** 画像の圧縮処理中に、**When** 保存ボタンを押す、**Then** 保存ボタンは無効化されており、圧縮完了まで送信されない（FR-016）

---

### User Story 3 - 日本語／英語の切り替え (Priority: P3)

ユーザーがアプリの表示言語を日本語または英語に切り替えることで、異なる言語圏のユーザーや、英語のフレーバー用語で記録したいユーザーがアプリを快適に利用できる。

**Why this priority**: 国際的なウィスキー愛好家コミュニティへの対応と英語フレーバー表現のサポートは重要だが、記録・画像機能が揃ってから実装しても遅くない。

**Independent Test**: 言語設定を日本語 → 英語に切り替え、全ての UI ラベル・プレースホルダー・エラーメッセージが英語に切り替わることで検証可能。

**Acceptance Scenarios**:

1. **Given** アプリを初回起動したとき、**When** 端末の言語設定を参照する、**Then** 対応言語（日本語・英語）であれば自動的にその言語で表示される（未対応言語は英語にフォールバック）
2. **Given** 日本語で表示中のアプリで、**When** 設定から「English」を選択する、**Then** アプリ全体の UI ラベルが英語に切り替わり、設定は次回起動後も保持される
3. **Given** 言語を英語に切り替えた状態で、**When** テイスティングフォームの項目を参照する、**Then** 外観・香り・味わい・余韻などのラベルが英語（Appearance / Nose / Palate / Finish）で表示される
4. **Given** ユーザーが入力したテイスティングコメント（日本語テキスト）がある状態で、**When** 表示言語を英語に切り替える、**Then** ユーザーが入力したテキストはそのまま保持される（UI ラベルのみ翻訳）

---

### Edge Cases

- オフライン状態での全操作（記録・画像登録・言語切り替え）が正常に動作するか？
- カメラ・フォトライブラリへのアクセス権限を拒否した場合、適切なメッセージを表示できるか？
- 画像ファイルが破損していた場合、ノート自体は保存できるか？
- 大量のノート（500件以上）が存在する場合、一覧表示のパフォーマンスは維持されるか？
- フォームの途中でアプリをバックグラウンドに移した場合、下書きとして自動保存され、再起動後に復元される。
- 下書きが存在する状態で新規作成を開始した場合、前の下書きは警告なしに破棄される。
- 画像圧縮処理中に保存ボタンを押した場合、圧縮完了を待たずに遷移してはならない（FR-016）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、銘柄名・蒸留所・ヴィンテージ・外観・香り・味わい・余韻・総合評価（0–100点）・フリーテキストメモを含むテイスティングノートの新規作成・編集・削除をサポートしなければならない
- **FR-002**: 銘柄名はノートの必須項目とし、未入力のままでは保存できないようにしなければならない
- **FR-003**: ユーザーは端末カメラでの撮影またはフォトライブラリからの選択により、ボトル画像をノートのサムネイルとして登録できなければならない
- **FR-004**: 登録する画像は端末上で自動圧縮・リサイズされ、保存サイズは 500KB 以下に収めなければならない
- **FR-005**: ノート一覧には、サムネイル画像・銘柄名・評価・記録日時を表示し、記録日時の降順（最新順）で並べなければならない
- **FR-006**: アプリは日本語と英語の 2 言語に対応し、全ての UI テキスト（ラベル・プレースホルダー・エラーメッセージ）が切り替わらなければならない
- **FR-007**: 初回起動時は端末のロケール設定に基づいて表示言語を自動選択し、未対応言語の場合は英語にフォールバックしなければならない
- **FR-008**: 言語設定はアプリ設定画面から明示的に変更でき、設定は次回起動後も保持されなければならない
- **FR-009**: ユーザーが入力したテイスティングコメント等のテキストは、表示言語を変更しても変更されてはならない
- **FR-010**: 全ての機能はネットワーク接続なしで動作しなければならない（オフラインファースト）
- **FR-011**: デザインはダークテーマ（黒・深みのあるネイビー）に金色アクセントを用いた高級感のある視覚表現を採用しなければならない
- **FR-012**: ノート作成・編集フォームへの入力内容は自動的に下書き保存され、アプリ再起動後も復元できなければならない。下書きは常に1件のみ保持し、新規作成開始時に前の下書きを上書きする
- **FR-013**: 外観・香り・味わい・余韻・メモの各テキストフィールドは、それぞれ最大 1,000文字を上限とし、超過時はユーザーに通知しなければならない
- **FR-014**: 同一銘柄名のノートは一意性制約なく何件でも作成できなければならない
- **FR-015**: ノートの削除操作時は確認ダイアログを表示し、ユーザーが承認した場合のみ削除を実行しなければならない
- **FR-016**: 画像圧縮処理中は保存ボタンを無効化し、圧縮完了前にフォームが送信されてはならない

### Key Entities

- **TastingNote（テイスティングノート）**: ウィスキー 1 銘柄に対する 1 回のテイスティング記録。属性: 銘柄名（必須・一意性制約なし）・蒸留所・ヴィンテージ・外観（最大1,000字）・香り（最大1,000字）・味わい（最大1,000字）・余韻（最大1,000字）・総合評価（0–100）・メモ（最大1,000字）・記録日時・最終更新日時
- **BottleImage（ボトル画像）**: ノートに紐付くサムネイル画像。端末上で 500KB 以下に圧縮・保存され、ノートと 1:1 の関係を持つ
- **Draft（下書き）**: 保存前の入力中ノート。アプリ終了・バックグラウンド移行時に自動保存される。同時に存在できるのは1件のみで、新規作成開始時に上書きされる
- **AppSettings（アプリ設定）**: アプリ全体の設定を保持するシングルトン。属性: language（`'ja'` または `'en'`）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは新規テイスティングノートをゼロから 2 分以内に作成・保存できる
- **SC-002**: ボトル画像の選択からノートへの登録完了まで 30 秒以内に完了できる
- **SC-003**: 500 件のノートが存在する状態でも、一覧画面の初期表示が 1.5 秒以内に完了する
- **SC-004**: 初めてアプリを使うユーザーの 90% が、説明なしでノートの作成と画像登録を完了できる
- **SC-005**: 言語切り替えが設定変更から 1 秒以内に全 UI に反映される
- **SC-006**: ネットワーク未接続状態でも、記録・閲覧・編集・削除の全操作が正常に動作する

## Assumptions

- テイスティングノートはデバイスローカルにのみ保存し、クラウドバックアップや他デバイスとの同期は本機能のスコープ外とする
- フレーバーホイール（風味タグの選択 UI）は本機能のスコープ外とし、フリーテキスト入力で対応する
- テイスティングノートの共有・エクスポート機能は本機能のスコープ外とする
- 画像は 1 ノートにつき 1 枚（ボトルのサムネイル）のみとする
- 評価スコアは 0–100 の整数とする（Jim Murray's Whisky Bible スタイルを参考に）
