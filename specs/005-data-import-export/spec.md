# Feature Specification: データ インポート・エクスポート機能

**Feature Branch**: `005-data-import-export`
**Created**: 2026-03-12
**Status**: Draft
**Input**: User description: "インポート・エクスポート機能の追加"

## Clarifications

### Session 2026-03-12

- Q: ボトル写真をエクスポートファイルに含めるか？ → A: 含める（完全バックアップ）
- Q: インポート実行前に確認ダイアログを表示するか？ → A: 確認あり（「〇件のノートを追加します。続行しますか？」）
- Q: パフォーマンス基準（SC-001/SC-002）は写真あり・なしを区別するか？ → A: 区別しない（基準は写真なしの場合のみ、写真ありはローディング表示で対応）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - テイスティングノートのエクスポート (Priority: P1)

ユーザーが自分のテイスティングノートデータを手元のデバイスにバックアップしたい。設定画面から「エクスポート」を実行すると、全ノートのデータをJSONファイルとしてダウンロードできる。データを失うことへの不安なく、デバイス変更やアプリデータ消去に備えられる。

**Why this priority**: データ損失リスクに対するユーザーの根本的な不安を解消する。インポートより先に実装すべき出力側の機能であり、単独でバックアップ手段として価値がある。

**Independent Test**: エクスポートボタンを押してJSONファイルがダウンロードされ、全ノートの情報が含まれていることを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** テイスティングノートが1件以上存在する状態で、**When** 設定画面の「データをエクスポート」を押すと、**Then** 全ノートのデータを含むJSONファイルがデバイスに保存される
2. **Given** ノートが0件の状態で、**When** 「データをエクスポート」を押すと、**Then** 空の配列を含むJSONファイルがダウンロードされる（エラーにならない）
3. **Given** エクスポートが完了したとき、**Then** ファイル名には日付が含まれ（例: `whisky-note-export-20260312.json`）、保存場所が分かりやすい
4. **Given** エクスポート中に問題が発生したとき、**Then** ユーザーにエラーメッセージが表示される

---

### User Story 2 - テイスティングノートのインポート (Priority: P2)

ユーザーが以前にエクスポートしたJSONファイルを読み込んで、ノートを復元したい。新しいデバイスへの移行や、誤って削除したデータの復元に対応できる。

**Why this priority**: エクスポートと合わせてバックアップ・リストアのサイクルが完成する。エクスポートが先に必要なため P2。

**Independent Test**: エクスポートで作成したJSONファイルをインポートし、元のノートが再表示されることで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** エクスポートで作成したJSONファイルを選択したとき、**When** 「インポート」ボタンを押すと、**Then** 「〇件のノートを追加します。続行しますか？」という確認ダイアログが表示される
2. **Given** 確認ダイアログで「続行」を選択したとき、**Then** ファイル内の全ノートがアプリに追加される
3. **Given** 確認ダイアログで「キャンセル」を選択したとき、**Then** データは変更されずにダイアログが閉じる
4. **Given** 既存のノートがある状態でインポートしたとき、**Then** 既存データを保持しつつ、ファイル内のノートが追加される（マージ動作）
5. **Given** 同じIDのノートが既に存在する場合、**Then** ファイル内のデータで上書きせず、スキップする（重複防止）
6. **Given** 無効なファイル（JSON形式でない、または対応フォーマットでない）を選択したとき、**Then** ユーザーに分かりやすいエラーメッセージを表示する
7. **Given** インポートが正常完了したとき、**Then** 追加されたノートの件数をユーザーに通知する

---

### Edge Cases

- ノートが0件の状態でエクスポートすると空データがダウンロードされる（エラーにしない）
- 破損したJSONファイル、または対応フォーマット以外のファイルをインポートした場合はエラーを表示してデータを変更しない
- インポートファイルのバージョンが現在のアプリと互換性がない場合はエラーを表示する
- 非常に多くのノート（例: 500件超）をインポートする場合でも処理が完了し、UIがフリーズしない
- インポートが途中で失敗した場合、部分的に追加されたデータは保持される（ロールバックせず、成功した分だけ追加）
- ボトル写真が添付されているノートをエクスポートする場合、写真データも含める（完全バックアップ）。インポート時も写真を復元する。ファイルサイズが大きくなる可能性（100件で最大50MB）があるため、エクスポート前にユーザーへの目安サイズ表示は行わないが、処理中はローディング表示を維持する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは設定画面に「データをエクスポート」ボタンを提供しなければならない
- **FR-002**: エクスポート時、全テイスティングノートのデータ（銘柄名・蒸留所・ヴィンテージ・外観・香り・味わい・余韻・評価・メモ・記録日時、およびボトル写真）をJSON形式でダウンロードできなければならない
- **FR-003**: エクスポートファイル名には実行日付を含まなければならない（例: `whisky-note-export-YYYYMMDD.json`）
- **FR-004**: エクスポートファイルにはフォーマットバージョン情報を含まなければならない（将来の互換性のため）
- **FR-005**: システムは設定画面に「データをインポート」ボタンを提供しなければならない
- **FR-006**: インポート時、ユーザーはデバイス上のJSONファイルを選択できなければならない
- **FR-007**: インポートは既存データを保持しつつ新しいノートを追加する（マージ動作）。既存と同一IDのノートはスキップする
- **FR-008**: インポート完了後、追加されたノートの件数とスキップされた件数をユーザーに通知しなければならない
- **FR-009**: 無効なファイル（不正なJSON、未対応フォーマット、未対応バージョン）をインポートした場合、エラーメッセージを表示してデータを変更してはならない
- **FR-010**: エクスポート・インポート中にエラーが発生した場合、ユーザーに分かりやすいエラーメッセージを表示しなければならない
- **FR-011**: インポート処理はUIをブロックせず、処理中はローディング状態を表示しなければならない
- **FR-012**: インポート実行前に「〇件のノートを追加します。続行しますか？」という確認ダイアログを表示しなければならない。キャンセル時はデータを変更しない

### Key Entities

- **エクスポートファイル**: JSONファイル。フォーマットバージョン、エクスポート日時、ノート配列を含む
- **ノートレコード**: テキストフィールド一式（brandName, distillery, vintage, appearance, nose, palate, finish, rating, notes, createdAt, updatedAt）とID、およびボトル写真データ（任意）
- **インポート結果**: 追加件数・スキップ件数・エラー情報

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: エクスポート操作を開始してからファイルがダウンロードされるまで、写真なし100件で3秒以内に完了する（写真ありの場合はローディング表示で対応）
- **SC-002**: インポート操作を開始してから完了通知が表示されるまで、写真なし100件で5秒以内に完了する（写真ありの場合はローディング表示で対応）
- **SC-003**: エクスポートしたファイルをインポートすると、全てのテキストデータが元の状態と一致して復元される（データ完全性100%）
- **SC-004**: 無効なファイルをインポートしようとした場合、既存のノートデータが変更されない（データ安全性100%）
- **SC-005**: エクスポート・インポートの操作フローを初めて使うユーザーの90%が、説明なしで完了できる（直感的なUI）

## Assumptions

- エクスポート形式はJSONのみ（CSV等は対象外）
- インポートはこのアプリ自身がエクスポートしたファイルのみを対象とする（他アプリからのインポートは対象外）
- ボトル写真はエクスポートに含める（完全バックアップ）。ファイルサイズは大きくなる可能性がある
- インポート時の重複判定はノートのIDベースで行う
- エクスポート・インポートはオフライン環境でも動作する（ローカル処理のみ）
- 大量データ（500件超）の処理も想定するが、ローカル処理のため外部通信は不要
